# 진료기록부 양식·운영 규칙 (record-format)

> 주치의의 본질은 환자 이력을 아는 것 — 기록이 없으면 매번 초진이다.
> 위치: `<프로젝트>/.project-doctor/records/` · 모든 파일 UTF-8(BOM 없음).

## 1. 최초 생성 게이트 (반드시 승인 후)

`.project-doctor/` 폴더가 없으면, 진단 완료·보고서 전달 **후에** 아래 문구로 묻는다:

> 진단 기록을 프로젝트 안 `.project-doctor/` 폴더에 남겨둘까요? 기록이 있으면 다음 검진부터 "지난번보다 좋아졌는지"를 비교해 드립니다.
> 알아두실 것: ① 이 폴더에는 프로젝트의 문제점 목록이 쌓입니다. ② 압축·복사로 프로젝트를 공유하면 git과 무관하게 이 폴더도 따라갑니다 (공유 전 제거 권장 — 공개 전 검진이 점검해 줍니다). ③ 승인하시면 앞으로 검진·치료 후 자동으로 기록합니다.

- 승인 → 폴더 생성 + 폴더 안에 `*` 한 줄짜리 `.gitignore`를 함께 생성 (자기 자신을 git에서 제외 — **프로젝트의 .gitignore는 건드리지 않는다**). 이후 방문부터는 자동 기재.
- 거부 → 기록 없이 진행. "기록이 없어 다음 검진에서 비교를 해드릴 수 없습니다" 한 줄 안내. 다음 검진 때 다시 물을 수 있다.

## 2. 기록 파일 양식

파일명: `records/<YYYYMMDD-HHmmss>-<모드>.md` (예: `20260610-143052-checkup.md`) — 같은 초에 충돌하면 끝에 `-2`, `-3` 연번을 붙인다.

```markdown
# 진료 기록 — <YYYY-MM-DD HH:mm>
- 스킬 버전: <SKILL.md 상단의 현재 버전> · 모드: <checkup/--deep/intro/pivot/release-check>
- 대상: <프로젝트 폴더 경로 — 사용자 홈 아래면 `~`로 축약해 기재 (개인정보 최소화)> · 분석 파일 수: N개
- 발견ID: DUP-01 🔴, BIG-03 ⚪, ...   ← 보고서 기계 판독 블록의 **기록부 확장 형식** (ID 뒤 심각도 — 다음 검진의 "악화" 판정에 필요)
- 영역 등급: 구조 B · 정리 C · 문서 D · 테스트 C · 안전 B · 종합 C
- 처치 내역: <실행한 항목 ID + git 커밋 해시 또는 백업 경로 + 되돌리기 명령> (없으면 "없음")
- 상태 요약: <한 줄>
```

## 3. 기재 금지 (절대 규칙)

- 비밀키·토큰·비밀번호는 **위치든 내용이든 기재 금지** (발견 사실은 보고서(채팅)로만 전달)
- 이메일·전화번호 등 개인정보 기재 금지
- 코드 원문을 길게 복사해 넣지 않는다 (위치와 ID만)

## 4. 재검진 비교 절차

1. 검진 시작 시 `records/`에서 **같은 모드의 최신 기록**을 읽는다 (기록부가 있을 때만 — 모드가 다른 기록과는 비교하지 않는다). 이전 기록에 발견ID 확장 형식(ID+심각도)이 없으면(구버전·손상) 비교를 생략하고 보고서에 사유 한 줄만 적는다
2. 이전 발견ID 집합 ↔ 이번 발견ID 집합을 대조:
   - **해결됨**: 이전에 있고 지금 없음
   - **그대로**: 양쪽 모두 있음
   - **신규**: 이번에 처음 나옴
   - **악화**: 같은 ID인데 심각도가 올라감 (⚪→🟡, 🟡→🔴)
3. 결과를 보고서의 비교 블록에 채운다 — 위치는 report-template 4절(표지→종합 판정→부위별 소견·오늘의 처방 다음)을 따른다. "악화" 판정은 이전 기록의 ID별 심각도(확장 형식)와 대조한다
4. 이전 기록과 스킬 버전이 다르면 비교 블록에 "(스킬 버전이 달라 항목 기준이 일부 다를 수 있음: vX.Y → vX.Z)" 한 줄을 덧붙인다

## 5. 기록 시점 (무변경 원칙과의 관계)

기록 쓰기는 **진단·보고가 모두 끝난 뒤** 마지막에 수행한다. "진단 중 무변경" 검증(시나리오 5)의 측정 시점은 "진단 종료 직후·기록 쓰기 전"이므로 충돌하지 않는다.
