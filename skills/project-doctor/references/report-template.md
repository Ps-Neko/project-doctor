## 공통 골격 (모든 모드의 보고서가 이 순서를 따른다)

### 1) 표지

제목 줄은 마크다운 헤딩으로 살아 있어야 하므로 **코드블록 밖**에 쓴다:

```markdown
# 🏥 프로젝트 건강검진 결과지 — <프로젝트 이름>
```

이어서 표지 정보 2줄을 코드블록으로:

```
환자: <프로젝트 이름> · 검진일: <일시> · 분석 파일: N개 (제외 목록 적용 후 — 자체 테스트 시 정답지도 제외)
스킬 버전: <SKILL.md 상단의 현재 버전> · 모드: 건강검진 <git 프로젝트면 (커밋 abc1234)>
```

모드별 표지 제목 (첫 줄의 명칭만 교체):

| 모드 | 표지 제목 |
|------|----------|
| 건강검진 | 프로젝트 건강검진 결과지 |
| 초진 차트 | 프로젝트 초진 차트 |
| 방향전환 | 프로젝트 방향전환 계획서 |
| 공개 전 검진 | 공개 전 검진 결과지 |

> 버전의 단일 출처는 SKILL.md — 이 템플릿에 버전 실값을 적지 않는다.

재검진(같은 모드의 이전 기록이 1건 이상 = **2회째 방문부터**)이면 표지 코드블록 끝에 옵션 1줄을 덧붙인다:
`등급 추이: 5월 D → 오늘 C` (요약 — 상세 비교는 4절이 담당)

### 2) 종합 판정 (5초 안에 상태 파악 — checkup·release-check에서만, intro·pivot은 생략 규칙 적용)

```markdown
## 종합 판정

# 🔴 D — 치료가 필요해요

> 등급 읽는 법: 🟢 A 건강 · 🟢 B 양호 · 🟡 C 주의 · 🔴 D 치료 필요
> ※ 종합은 부위 평균이 아니라 발견 건수·심각도의 **합산 점수**입니다 — 부위 등급보다 낮을 수 있어요.

**의사 소견**: 작지만 손볼 곳이 골고루 있습니다. 돈 계산 코드 중복(심각 1)부터 잡는 걸 권합니다.
```

**등급↔표현 고정표** (이 표 밖의 표현 금지 — 회차마다 판정어가 바뀌면 신뢰가 깨진다):

| 등급 | 신호등 | 판정어 |
|------|--------|--------|
| A | 🟢 | 건강해요 |
| B | 🟢 | 양호해요 |
| C | 🟡 | 주의가 필요해요 |
| D | 🔴 | 치료가 필요해요 |

규칙:
- 등급 산정은 진단 카탈로그(checkup-catalog.md)의 "검진 등급 규칙" 절(심각도 가중 합산)을 따른다. "감"으로 등급을 매기지 않는다.
- **의사 소견은 종합 등급과 같은 방향의 톤** — 등급은 D인데 소견이 태평하거나, A인데 겁을 주면 안 된다. 1~2문장, 첫 처방 항목을 자연스럽게 가리킨다.

### 3) 부위별 소견 + 오늘의 처방

```markdown
## 부위별 소견

| 부위 | 판정 | 소견 |
|------|------|------|
| 구조 | 🟡 주의 (C) | 돈 계산 코드가 3곳에 복사되어 있어요 |
| 정리 | 🟡 주의 (C) | 백업·실험 파일이 그대로 남아 있어요 |
| 문서 | 🟡 주의 (C) | 설명서(README)도 부품 목록도 없어요 |
| 테스트 | 🟢 양호 (B) | 자동 확인 장치는 없지만, 규모가 작아 당장 위험하진 않아요 |
| 안전 | 🟡 주의 (C) | 이 컴퓨터 전용 폴더 주소가 코드에 박혀 있어요 |

## 오늘의 처방

진단 결과: 🔴 심각 N건 · 🟡 권장 N건 · ⚪ 참고 N건
👉 **당장 1건만 하신다면**: 심각 1(돈 계산 코드 중복) — 지금 "심각 1 실행해줘"라고 하시면 제가 고칩니다.
📅 **재검 권고**: <YYYY-MM-DD> (등급 고정표 기준 — A 90일 / B 60일 / C 30일 / D 14일 뒤)
```

규칙:
- **소견의 톤은 판정과 같은 방향** — 🟢 부위에 발견이 있으면 "~지만 당장 위험하진 않아요"처럼 판정 쪽으로 마무리한다. 판정과 소견이 반대 방향이면 읽는 사람이 혼란에 빠진다 (예: "B" 옆에 "하나도 없습니다"라고만 쓰지 않는다).
- 부위 판정 칸은 신호등+판정어+괄호 등급 — 등급↔표현 고정표를 따른다.
- "당장 1건"은 8절의 숙제 선정 규칙과 **같은 항목** (심각도 순 → 인벤토리 순, 상위 1건).
- `진단 결과:` 줄의 건수 표기는 유지 (구 "요약 한 줄" 계약).
- 지난 숙제가 미이행이면 복귀 문안(record-format 4장 6번)을 "당장 1건" 줄 **바로 앞에** 넣는다 — **책망 금지**.

### 4) (재검진일 때만) 이전 기록 대비 비교

```markdown
## 지난 검진과 비교 (기준: <이전 기록 일시>, 스킬 <이전 버전>)
| 구분 | 건수 | ID |
|------|------|-----|
| ✅ 해결됨 | N | ... |
| ➖ 그대로 | N | ... |
| 🆕 신규 | N | ... |
| ⬆️ 악화 | N | ... |
```

판정 기준과 절차는 `record-format.md` 4장을 따른다. 기록부가 없으면 이 절은 생략. 0건인 행의 ID 칸은 `(없음)`으로 적는다.
비교표 **위에** 처치 경과 1줄을 먼저 보여준다: `처치 부위 경과: [DUP-01] 재발 없음` — 판정: 직전 같은 모드 기록의 "처치 내역" ID가 이번 발견ID에 있으면 **재발**, 없으면 **재발 없음** (기존 필드만으로 도출). 직전 기록의 처치 내역이 "없음"이면 이 줄은 생략한다.

### 5) 발견 항목 본문 — 항목마다 카탈로그 ID + 4요소 의무 (제목의 ID + 본문 4줄)

```markdown
### 🔴 심각 1 [DUP-01] — 같은 기능의 코드가 3곳에 복사되어 있음
- **무슨 뜻인가요?** 한 곳을 고치면 나머지 2곳도 따로 고쳐야 해서, 빠뜨리면 버그가 됩니다.
- **어디?** `scripts/a.py`, `scripts/b.py`, `tools/c.py`
- **고치면?** 한 곳만 고치면 전체에 반영됩니다. 예상 작업: 파일 1개 신설, 3개 수정.
- **승인 명령:** "심각 1 실행해줘"
```

규칙:
- 우선순위는 🔴 심각 / 🟡 권장 / ⚪ 참고 **3단계만** 사용
- 제목에 반드시 카탈로그 ID(`[DUP-01]`) 포함
- 전문 용어 옆에는 일상 언어 풀이를 붙인다

### 6) 발견이 0건일 때 (빈손 방지 양식)

```markdown
## 오늘의 처방

진단 결과: 발견된 문제 없음 🎉
(종합 판정·부위별 소견은 동일하게 출력 — 종합은 🟢)
**잘 유지되고 있는 점** (2~3개):
- 예: 모든 함수가 한 화면 안에 들어오는 크기로 유지되고 있습니다.
- 예: README만 보고 실행 방법을 알 수 있습니다.
```

### 7) 발견이 너무 많을 때 (압도 방지 규칙)

발견이 10건을 초과하면 — 본문에는 **심각 전부 + 권장 상위 5건**만 싣고, 나머지는 보고서 끝 "부록: 나머지 발견 항목"에 ID·제목 한 줄씩만 나열한다.
- "상위" 기준: 심각도(🔴→🟡→⚪) 순, 동률이면 카탈로그 인벤토리 순 — **부록의 나열 순서도 같은 기준**
- 정밀 검진의 신규 발견(HIST·DEP 등)은 근거 설명이 중요하므로 부록으로 밀지 않고 본문에 싣는다
- **재발 항목**(처치 경과 "재발" 판정)도 부록으로 밀지 않고 본문에 싣는다

### 8) 다음 진료 안내 (v2.1에서 처방 줄을 3절 "오늘의 처방"으로 이동 — 절 위치·순서는 동결 유지)

```markdown
## 다음 진료 안내
- 다음 검진은 언제쯤이 좋을까요? (권장: <YYYY-MM-DD> — 3절 재검 권고와 같은 날짜) ← 사용자가 답하면 그 날짜를 기록부에 기재
- 연계 추천: 공개 예정이라면 release-check를 받아보세요.
비교: 해결 1 / 그대로 12 / 신규 2 / 악화 0
숙제: HARD-01
```

규칙:
- **숙제 선정**: 심각도 순(🔴→🟡→⚪), 동률이면 카탈로그 인벤토리 순 — 상위 **1건만**. 심각 0건이면 권장 최상위 1건, 발견 0건이면 `숙제: (없음)` — 3절 "당장 1건"과 같은 항목이어야 한다
- **첫 검진**(이전 기록 없음)이면 `비교:` 줄은 생략하고 **그 자리에** 예고 1줄을 넣는다: "다음 검진부터는 오늘과 비교한 추이가 표시됩니다"
- `비교:` `숙제:` 줄은 기계 판독용 — 형식 고정(한 줄, 슬래시·쉼표 구분), 이 절 안에만 위치 (9절 발견ID 블록이 보고서 맨 마지막 줄이라는 계약은 불변)
- **후기 고리**: 단골 차트(추이 줄)가 처음 출력된 보고서에서만, 이 절 끝에 후기 요청 1줄 — 기록부의 "후기 요청 이력"이 '예'면 절대 반복하지 않는다

### 9) 기계 판독용 발견ID 블록 (보고서 맨 마지막 줄 — 채점 스크립트가 읽는다)

```
발견ID: DUP-01, BIG-03, DOC-02
```

- 발견 0건이면: `발견ID: (없음)`
- 이 블록에는 본문·부록 구분 없이 **이번 검진의 모든 발견 ID를 빠짐없이** 나열한다 (7)절에 따라 부록으로 밀린 항목 포함)
- 나열 순서는 카탈로그 인벤토리 순서를 따른다
- 이 줄은 사람용이 아니라 기계 대조용 — 형식(쉼표+공백 구분, 한 줄)을 절대 바꾸지 않는다

## 모드별 본문 양식

- **건강검진**: 위 공통 골격 그대로 (ID+4요소 발견 항목)
- **초진 차트** (해설형 — 4요소 대신): 본문 = 한 줄 소개 → 프로젝트 지도(핵심 파일 5개 표) → 실행·중지 방법(추정 표기) → 만지면 위험한 곳 → 미완성·방치 구간. 상세 양식은 `intro-chart.md`. 표지·발견ID 블록은 공통 골격 유지, **종합 판정·부위별 소견은 생략**("정밀 등급은 checkup에서" 한 줄로 대체), 인용 ID 없으면 `발견ID: (없음)`
- **방향전환**: 본문 = 갈림길 비교표(해당 시) → 유지/수정/폐기 분류표 → 마일스톤(멈춰도 동작 정의) → 폐기 2단계 → 완료 체크리스트. 상세 양식은 `pivot-playbook.md`. 인용 ID 없으면 `발견ID: (없음)`
- **공개 전 검진**: 공통 골격 + 오늘의 처방 직후에 **판정 블록** (통과/보류 + 검진 시점 의무 기재 + 한계 고지 3종). 상세 양식은 `release-checklist.md` 3장
