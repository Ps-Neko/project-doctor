# 프로젝트 주치의 (Project Doctor)

내 프로젝트가 마음에 안 들거나, 방향을 바꾸고 싶거나, 남에게 넘겨야 할 때 —
프로젝트를 진단하고, **승인한 개선만** 안전하게 실행해 주는 Claude Code 스킬입니다.

> **현재 버전: v2.1.0 "단골 주치의 + 결과지 개편"** — 전 모드 동작 + 단골 기능(등급 추이 · 치료 부위 경과 · 이번 주 처방 · 검진 주기 안내) + 건강검진 결과지 양식(종합 판정·부위별 소견·오늘의 처방). 두 번째 방문부터 "지난번보다 좋아졌는지"를 아는 주치의가 됩니다. 변경 이력: [CHANGELOG.md](./CHANGELOG.md)

## ⚠️ 사용 전 꼭 알아두세요 (필수 고지)

1. **진단 시 프로젝트 내용이 Claude(Anthropic) 서버로 전송됩니다.** 이것은 Claude Code의 기본 동작입니다. 회사 기밀·고객 데이터가 포함된 프로젝트라면, 회사의 AI 사용 정책을 먼저 확인하세요.
2. **비용**: 검진은 Claude 사용량(토큰)을 소모합니다. 정밀 검진(`--deep`)은 기본 검진보다 수 배 더 듭니다 — 평소엔 기본 검진을 권합니다.
3. **검사 범위와 면책**: 이 스킬은 진단 카탈로그에 정의된 알려진 패턴만 검사합니다. 보안 취약점 분석·법적 검토는 범위 밖이며, 테스트에서의 합격 기준(예: 탐지율 100%)은 내부 품질 기준이지 성능 보증이 아닙니다. 결과에 대한 최종 판단과 책임은 사용자에게 있습니다.

## 설치 (PowerShell)

1. 이 저장소(automation-main)를 내려받아 압축을 풉니다. — **공식 저장소의 릴리스에서만 받으세요** (메신저로 돌아다니는 zip 사본은 변조 위험이 있습니다)
2. PowerShell을 열고 `cd <받은 폴더 경로>`로 이동합니다. (예: `cd C:\Users\내이름\Downloads\automation-main`)
3. 아래 명령을 실행합니다.

```powershell
New-Item -ItemType Directory -Force ~/.claude/skills
Copy-Item -Recurse -Force skills/project-doctor ~/.claude/skills/
```

복사 후 **Claude Code를 새로 시작**해야 `/project-doctor`가 인식됩니다.
(재설치·업데이트할 때도 같은 명령을 그대로 다시 실행하면 됩니다 — 여러 번 실행해도 안전합니다.)
"경로를 찾을 수 없음" 오류가 나오면 2번의 폴더 위치를 다시 확인하세요.

## 사용법

```
/project-doctor                     # 접수 문진 — 질문 1~2개로 알맞은 모드 안내
/project-doctor checkup             # 건강검진 (기본 검진) — 진단 → 보고 → 승인한 항목만 치료
/project-doctor checkup "<경로>"     # 지정 폴더 검진 — 한글·공백 경로는 따옴표
/project-doctor checkup --deep      # 정밀 검진 — 기본 + git 이력 핫스팟·의존성 점검·AI 작업물 문진
/project-doctor intro               # 초진 차트 — "이게 뭔지부터" 해설 + 인수인계 소견서
/project-doctor pivot "<목표>"       # 방향전환 계획 — 목표가 없으면 갈림길 비교부터
/project-doctor release-check       # 공개 전 검진 — 비밀키·개인정보 점검 + 통과/보류 판정
```

## 동작 원칙 (안전장치)

1. **진단은 읽기 전용** — 보고서가 나오기 전에는 아무것도 고치지 않습니다
2. **승인한 것만 실행** — 보고서에서 고른 항목만, 한 번에 하나씩. 실행 전에 "무엇이 바뀌는지" 목록을 먼저 보여줍니다
3. **항상 되돌릴 수 있게** — 실행 전 체크포인트(git 또는 백업), 실행 후 **되돌리기 명령 한 줄**을 알려줍니다. git이 없는 폴더에서는 백업 방식을 안내하며, 되돌릴 방법이 없으면 실행하지 않습니다
4. **이력 기억** — 승인 시 `.project-doctor/` 폴더에 진료 기록을 남겨, 다음 검진부터 "지난번보다 좋아졌는지" 비교해 줍니다

## 자주 묻는 질문

- **검진이 내 코드를 망치지 않나요?** 진단 단계는 읽기만 합니다. 고치는 건 사용자가 항목을 승인했을 때만, 되돌리기 보장과 함께 진행됩니다.
- **큰 프로젝트도 되나요?** 파일 100개(자동 생성 폴더 제외)가 넘으면 핵심 항목만 빠르게 보거나 범위를 좁히도록 제안합니다.
- **`.project-doctor` 폴더는 뭔가요?** 진료 기록입니다. 지워도 스킬 동작에는 문제없지만 다음 검진에서 비교를 못 합니다. **프로젝트를 압축해 공유할 때는 이 폴더를 빼세요** (문제점 목록이 들어 있습니다).
