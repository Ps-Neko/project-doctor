# EXPECTED.md — <별칭> 정답지

이 파일은 "프로젝트 주치의" 스킬 자체 테스트의 정답지로, 검진 시 진단 대상에서 제외된다(SKILL.md 절대 규칙 9).

> 복사해서 `external-corpus/<별칭>/EXPECTED.md`로 채운다. `<...>`는 실제 값으로 바꾼다.
> 머리말 첫 줄의 "스킬 자체 테스트의 정답지" 표식은 **지우지 말 것** — 이 표식이 있어야
> 검진이 정답지를 읽지 않고(커닝 방지) 진단 대상에서 제외한다.

## 대조 절차

1. 검진 보고서 마지막 줄 `발견ID:` 에서 ID 목록을 추출한다.
2. `python tests/run_checks.py <보고서> <이 파일>` (저장소 루트에서 실행)
3. 탐지율 = (아래 "심은 문제" 중 보고서에 나온 ID 수) ÷ (총 건수).

## 심은 문제 (N건)

> 사람이 코드를 읽고 합의한 **실제 문제**만 적는다. ID 는 `references/checkup-catalog.md`의 것을 쓴다.
> '위치' 열에 파일명을 적으면 BL-22 위치 게이트가 "보고서가 그 파일을 가리키는지"도 확인한다.
